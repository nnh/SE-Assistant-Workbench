# Google Drive 権限一括エクスポートツール (TypeScript版)

Google ドライブの指定フォルダ配下を再帰的に走査し、ファイル・フォルダの共有権限、オーナー、ショートカット情報などをスプレッドシートへ抽出するツールです。

## 🚀 主な機能

- **再帰的スキャン**: 指定したルートフォルダ配下の全階層を自動走査。
- **権限の可視化**: オーナー、編集者、閲覧者、アクセス種別（リンク共有状態など）を一覧化。
- **ショートカット解析**: ショートカットファイルのリンク先（ターゲットID）を特定。
- **高度な除外設定**: 「対象外フォルダ」シートにIDを記入することで、巨大なディレクトリのスキャンをスキップ。
- **レジューム機能**: 「検索済み」シートに処理済みIDを記録。中断しても続きから再開可能。
- **堅牢な設計**: TypeScriptによる型定義と、`safeGet_` によるエラーハンドリングを実装。

## 📁 プロジェクト構成

Aside（clasp）環境での開発に最適化された構成です。

- `src/configService.ts`: 初期設定やプロパティ操作。
- `src/consts.ts`: シート名やプロパティキーなどの定数管理。
- `src/driveRepository.ts`: ドライブからのデータ抽出ロジック。
- `src/index.ts`: エントリーポイント（GASから実行する関数）。
- `src/permissionService.ts`: 再帰探索やバッチ処理のメインロジック。
- `src/setup.ts`: スプレッドシートのシートセットアップロジック。
- `src/transformer.ts`: 「共有権限」シートのデータを元に、用途別の各シート（基本情報、アクセス種別、編集者、閲覧者）へ分割・出力するロジック。
- `src/utils.ts`: 共通ユーティリティ（エラーラップ関数など）。

## 📋 事前準備

### プロジェクトのデプロイ

- Googleドライブの任意のフォルダに新規スプレッドシートを作成してください。
- プロジェクトのルートフォルダに`.clasp-dev.json`と`.clasp-prod.json`を作成してください。

```
{
  "scriptId": "スクリプトIDをここに入れる",
  "rootDir": "dist",
  "parentId": "スプレッドシートのIDをここに入れる",
  "scriptExtensions": [
    ".js",
    ".gs"
  ],
  "htmlExtensions": [
    ".html"
  ],
  "jsonExtensions": [
    ".json"
  ],
  "filePushOrder": [],
  "skipSubdirectories": false
}
```

- ターミナルでプロジェクトのルートまで移動してください。`npm run deploy`で`.clasp-dev.json`に設定したスプレッドシートにスクリプト群がプッシュされます。

### 初期セットアップ

プロジェクトをデプロイ後、GASエディタまたはメニューから `initializeProject_` 関数を実行してください。既存のシートはすべて削除され、以下のシートとスクリプトプロパティの雛形が自動生成されます。

#### 1. 生成されるシート一覧

| シート名                   | 役割                                                         |
| :------------------------- | :----------------------------------------------------------- |
| **共有権限**               | 全スキャンの抽出結果が出力されるメインシート。               |
| **検索済み**               | 処理済みIDが記録されます。タイムアウト時の再開に使用します。 |
| **検索対象外フォルダ**     | **A列**に走査をスキップしたいフォルダIDを入力してください。  |
| **基本情報**               | ファイル名やオーナー、IDなどの基本データを抽出。             |
| **アクセス種別**           | リンク共有設定（公開範囲）の情報を抽出。                     |
| **編集者リスト**           | 外部ドメインの編集権限を持つユーザーを抽出。                 |
| **閲覧者リスト**           | 外部ドメインの閲覧権限を持つユーザーを抽出。                 |
| **外部共有アイテムリスト** | 外部共有が発生しているアイテムの統合リスト。                 |
| **外部共有フォルダリスト** | 外部共有されているフォルダのみのリスト。                     |
| **外部共有ファイルリスト** | 外部共有されているファイルのみのリスト。                     |

#### 2. スクリプトプロパティの設定

セットアップ実行後、Google Apps Script の「プロジェクトの設定」にて以下のプロパティ値を設定してください。

| キー                             | 内容                                                 |
| :------------------------------- | :--------------------------------------------------- |
| **`TARGET_ROOT_FOLDER_ID`**      | スキャンの起点となるフォルダのID。                   |
| **`DESTINATION_ROOT_FOLDER_ID`** | フォルダ・ファイルの移動先となるルートフォルダのID。 |

> [!CAUTION]
> `initializeProject_` を実行すると、スプレッドシート内の既存シートはすべて削除されます。必要なデータがある場合は事前にバックアップを取ってください。

## 📖 使い方

### `TARGET_ROOT_FOLDER_ID`で指定したIDのフォルダ配下のフォルダ・ファイル一覧出力

1.  **除外設定**: スプレッドシートの「対象外フォルダ」シートに、スキャン不要なフォルダIDを登録します。
2.  **実行**: `exportFolderPermissionsRecursive` 関数を実行します。
3.  **確認**: 実行ログに `✅ ◯件処理完了` と表示され、スプレッドシートにデータが蓄積されます。
4.  **再開**: タイムアウト等で終了した場合は、そのまま再度 `exportFolderPermissionsRecursive` を実行すれば「検索済み」シートにない項目から自動的に再開します。

## 📊 出力項目の詳細

| 項目                   | 説明                                                     |
| :--------------------- | :------------------------------------------------------- |
| **タイプ**             | 「フォルダ」または「ファイル」                           |
| **パス**               | ルートフォルダからの相対パス                             |
| **名前**               | アイテムの名称                                           |
| **ID / URL**           | Googleドライブ上の固有IDと直リンク                       |
| **アクセス種別**       | リンク共有設定（ANYONE, DOMAIN, LIMITEDなど）            |
| **オーナー**           | 所有者のメールアドレス                                   |
| **編集者 / 閲覧者**    | 各権限を持つユーザー一覧（改行区切り）                   |
| **ショートカット元ID** | ショートカットの場合、その参照先（ターゲット）のIDを表示 |

## ⚠️ 注意事項

- **実行制限**: GASの30分制限を超えると停止します。その場合は再度実行してください。
- **権限不足**: 権限の取得に失敗した場合、失敗した項目は「!取得不可!」と表示されます。
- **シートのクリア**: 新しくスキャンを最初からやり直したい場合は、「共有権限」と「検索済み」シートのデータを削除してから実行してください。

### 外部共有フォルダ・ファイル一覧の出力

1. `splitPermissionData`関数を実行します。「共有権限」シートの情報から、外部メールアドレスとの共有または「リンクを知っている全員」がアクセスできる設定を行っているフォルダ・ファイルのみ抽出して「外部共有アイテムリスト」に出力します。さらに、タイプが「フォルダ」の行のみ「外部共有フォルダリスト」シートに出力します。
